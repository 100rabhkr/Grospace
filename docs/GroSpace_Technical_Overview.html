1. System Overview

GroSpace is an AI-powered commercial real estate lease management platform designed for multi-brand retail operators managing portfolios of 50–500+ outlets across India. It automates lease data extraction, obligation tracking, payment management, risk analysis, and portfolio intelligence.

What It Does
  • AI extraction of 60+ structured fields from lease PDFs (including scanned/handwritten)
  • Automated obligation + alert generation from extracted data
  • Deal pipeline (Kanban) for tracking outlet acquisition stages
  • Payment tracking with auto-generation from obligations
  • Portfolio analytics with rent/sqft, rent-to-revenue ratios
  • Smart AI chat for natural language portfolio queries
  • Multi-org, role-based access with Supabase RLS
Key Metrics
  • 48 API endpoints
  • 18 frontend pages
  • 41 frontend API functions
  • 60+ extracted lease fields
  • 11 alert types
  • 6 deal pipeline stages
  • 8 risk flag categories
  • 4 rent models supported

State Management

All pages are "use client" components using React hooks (useState, useEffect, useCallback, useRef). No global state store — each page fetches its own data via the API client. Optimistic updates are used for drag-and-drop (pipeline) and priority toggling.

API Client (src/lib/api.ts)

Centralized HTTP client with 41 functions. Auto-attaches Supabase JWT from browser session. All calls go through apiFetch() which handles auth headers, error parsing, and JSON response.

// Pattern used across all API functions
async function apiFetch(endpoint: string, options: RequestInit = {}) {
  const token = await getAuthToken();  // Supabase session
  const headers = { "Content-Type": "application/json", Authorization: `Bearer ${token}` };
  const response = await fetch(`${API_URL}${endpoint}`, { ...options, headers });
  if (!response.ok) throw new Error(error.detail);
  return response.json();
}

4. Backend Architecture

FastAPI Monolith Structure

Single main.py file (~3000 lines) containing all endpoints, schemas, helpers, and AI logic. Deployed as a Docker container on Railway with auto-deploy from GitHub.

Module Organization (within main.py)
  • Schemas (lines 58–290) — Pydantic models: extraction schema, request/response models
  • Activity Logger (lines 296–309) — Non-blocking audit trail for all mutations
  • Notification Routing (lines 316–480) — Channel determination, Resend email, MSG91 WhatsApp
  • Auth Middleware (lines 482–420) — JWT validation, user extraction, org scoping
  • Document AI Endpoints — /classify, /extract, /qa, /risk-flags, /upload-and-extract
  • Business CRUD — Agreements, outlets, alerts, payments, obligations, organizations
  • Pipeline & Showcase — Deal stages, drag-and-drop, public share tokens
  • Smart AI Chat — Portfolio-wide natural language queries via Gemini
  • Digest & Invite — Email digest collection + delivery, team invitation emails

Row-Level Security (RLS)

All tables have RLS enabled. Policies scope data by org_id matched against the authenticated user's profile. The backend uses service_role key to bypass RLS for system operations (confirm & activate, payment generation, alert scheduling).
